Healthcare Data Mining

This website gives us information about the data analytics and knowledge gathered from the dataset provided

The different task details and their functionality requirements are listed below

Task 1

Web Application creation

Task 1 involves the development of a methodology that is useful for the investigation of variations in a series of clinical and financial outcomes, in relation to any patient admission diagnosis code. The dataset you are going to use is the same dataset you used for Project 1. You need to develop a web application where the user is providing input of values for the following three attributes:

(i) Patient age group (ii) Gender (iii) Admission diagnosis code

Task 2

Classification of features

You are asked to use appropriate java libraries (readily available on java library resources on the web) to identify the most important features which are important towards the prediction of the discharge status. This is a feature selection process. Explain how the feature selection has been possible with the source code you have used. The features that have been returned back by this procedure will then be used to perform logistic regression analysis. The “LIBLINEAR -- A Library for Large Linear Classification” library can be your resource. The link below is going to provide to you the appropriate source code to proceed.

Task 3

Association Mining

Task 3 will involve a core association rule mining functionality. You are going to ask user to exclude features of no interest. That means that the user may manually uncheck the features which will not be used for the association rule mining. The user will also define specific support and confidence thresholds. The reduced feature set is going to be used for the calculation of support and confidence. The rules can only include 1 feature value in the “if” part of the rules. (We do not want to output rules like for example diagnosis_code_1=a, diagnosis_code_2=b  discharge destination=x. The results are going to be restricted, in accordance to the support and confidence intervals that the user has defined some moments ago.
